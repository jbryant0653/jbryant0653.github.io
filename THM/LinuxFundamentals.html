Commands

  • Unix was created by Ken Thompson and Dennis Ritchie in 1970 at AT&T Bell Lab.
    • Then GNU Project was made to make a bunch of open source software and tools for a UNIX like operating system. So Richard Stallman made a OS called GNU (Gnu Not Unix) in 1983.
      • A GNU license was released so that people could create these open source content.
    • Linux began in 1991 by Linus Torvalds and was combined with the GNU software to add onto the Linux kernel and this created the name GNU/Linux.
    • Linux is based on Unix and is used as an umbrella term to mean all OS's that are based on Unix.
  • Linux is used in things like: Websites, car control panels, checkout tills and registers, traffic light controllers.
  • Commands use -char syntax called flags or switches to do more detailed tasks.
    • Use --help flag to learn how to use the command tool.
    • Or use the MAN aka manual page to see the full documentation.
      • Syntax: man command
  • cd: Change Directory
    • cd . (current directory)
    • cd .. (Previous directory)
  • ls: List
    • ls -a (all)
    • ls -l (long format which lists permissions.)
      • ls -l fileName (will only do that file)
    • ls Directory (list the files in that directory)
  • cat: Concatenate
    • cat Directory/Path/File.txt (prints out that file)
  • pwd: Print Working Directory
  • echo: Print text
    • echo wowowo (prints the text)
  • whoami:
  • find: searches for a file or folder
    • find -name file (or *.txt)
  • grep: Comes from the ed editor which had a command called g/re/p which means Global/Regular Expression/Print which finally became its own utility. This searches within files for a specific pattern.
    • grep "regex pattern" File/Files
  • wc: Word Count
    • wc -l File/Files (lines)
    • wc -w File/Files (words)
    • wc -c File/Files (Bytes)
    • wc -m File/Files (characters)
    • wc -L File/Files (longest line)
  • touch: create empty file
    • touch filename
  • mkdir: MaKe DIRectory
    • mkdir DirectoryName
  • cp: CoPy files or folder
    • cp oldFile newFile
  • mv: MoVe or merge file or folder
    • mv oldFile NewFile (This is the same as cp)
    • mv oldFile ExistingFile (This will Merge the oldFile to the ExistingFile)
    • mv oldFile Directory/ (The / indicates it as directory and not a new file.)
  • rm: ReMove file or folder
    • rm fileName
    • rm -R DirectoryName
  • file: Determine file type
    • file fileName
  • wget: Download content from a webstie
    • wget website/file

Shell Operators [NOT DONE]

  • &
    - Run Command in the background of your terminal.

    Copying a large file
    • Use it to copy a large file that might take awhile to continue using the terminal.

  • &&

    • Combine multiple commands into one line. Order is from left to right.
    • The second command will only run when the first command finishes.

  • >
    - Takes the output of a command and redirects it somewhere else.

    Creating a new file using echo

    echo heyyyyyy > welcome

    • If the file already exists it will overwrite the content.

  • >>

    • Will append the content to the bottom of the file instead of overwriting it.

SSH and Remote Connection

  • SSH means Secure Shell and is a protocol to execute commands on a remote device that is properly encrypted.
  • A password is usually needed to connect to a remote device.
    Command structure

    ssh username@ipAddress

  • SCP: Secure Copy Files using SSH protocol to and from remote computer
    • Makes its own SSH session and so you don't have to be SSH'd into the system already.
    • scp localFile userName@ipAddress:newFile (Local to Remote)
    • scp userName@ipAddress:remoteFile localFile (Remote to Local)
      • Use -r for recursively copy a full directory
      • Use -P port to specify port (defualt 22)
      • Use -p to preserve file attributes.
    • Alternative format: scp localfile site.name.com:newfile
  • Python3 Webserver: start up a webserver to wget files from a remote machine.
    • python3 -m http.server
      - Default port is 8000
      Attention
      • Without configuration this can be dangerous! There is no authentication.
    • Secure Alternatives: Caddy, nginx, apache server.
      • You want to use HTTPS, SSL/TLS certs, and password protected.

File & Folder Permissions

  • Characteristics of a Item are: What actions are allowed and Who can perform the action.

    • File Ownership
    • File Permission

  • Every file has 3 kinds of owners: User, Group, Other

    • User: owner of the file.
    • Group: Every user is apart of a group and sometimes the group is called the same as your username.
      • Use the command groups to see what groups you belong to.
    • Other: A large group of every single user on the system.

  • Every file and directory has 3 permissions for each owner:

    • Files:

      • Read: view or copy file contents
      • Write: Modify file content
      • Execute: run the file if executable

    • Directories:

      • Read: list and copy names of all files from a directory
      • Write: Add or Delete files in a directory
      • Execute: Can enter the directory
      Directory RWE Precedence
      • Directory Execute must be On for Dir Write and all File permissions to work!
      • If Dir Read and Write are Off, you can still Read and Write to file contents if you know the File names!
        • Unless you have the sticky bit on "t" for the directory.

  • Use the stat command to get information about a files permissions or filesystem information.

    • stat fileName (about file)
    • stat -f fileName (about filesystem)

  • Or use the ls -l command for information about the files and folders.

    • Filetype:

      • "-": Regular File
      • "d": Directory
      • "l": Symbolic Link
        • File that points to another File, Directory, non-existent files.
        • Create using: ln -s targetFile linkName
      • "c": Character Device File
        • Files that provide access to char oriented devices such as keyboards and mice or serial ports which sequentially handle information. The files provide programs access to these hardware device.
        • stored in /dev and are associated with device drivers which are shown as numbers when doing ls -l.
      • "b": Block Device File
        • Files that provide access to block oriented devices such as storage devices and are stored in /dev.
      • "p": Named Pipe (FIFO)
        • (First In First Out), This file acts a intermediate area for processes to inter-communicate between each other. The files are bidirectional and are read and written to.
        • unnamed or anonymous pipes are use by the | symbol and only exist when the processes are alive.
          • The named version has persistent storage in the /dev folder.
      • "s": Unix Domain Socket
        • Same as named pipe accept it is typically used for communicated between processes across a network or local but similar processes.

    • Permissions: The read,write, and execute permission for the user, group, and other.

        • "r": Read
        • "w": Write
        • "x": eXecute
          • "X": Changes only Directories which can be used for large scale chmoding to not change any files!
        • "-": no Permission
        • "s": SUID or SGID (Set User or Group ID).
          • This will make it where when others execute the file it executes it will the User or Group owners permissions.
          • "S": If the x is not on for user or group to show you that there is an error.
        • "t": Set sticky bit which makes it so that all files in a given directory can only be deleted or renamed by the file owners or root.
          • "T": If the x is not on for other to show that there is an error.

      • Changing Permissions using chmod command:

        • Two modes: Absolute and Symbolic

        • Absolute:

          • Each section(user,group,other) is represented using octal format which is a 7 digit system or 3 bit system.
            • The MSB or 3rd bit is the "Read" value, 2nd bit is the "Write", and LSB or 1st bit is "eXecute".
            • This means from binary the "r" value is 4 in decimal, "w" is 2, and "x" is 1, and "-" is 0.
          • So in total each section will be a number 0 to 7 and make 3 digit number to give to chmod.
          chmod absolute mode
          • chmod 356 fileName
            • User: 3 = 2+1 so Write and eXecute. Or 3 => 011 which is -wx
            • Group: 5 = 4+1 so Read and eXecute. Or 5 => 101 which is r-x
            • Other: 6 = 4+2 so Read and Write. Or 6 => 110 which is rw-
          Special Permissions!
          • "s" SUID is 4, "s" SGID is 2, and "t" is 1
          • Use the optional 4th digit to set these values.
          • chmod 7356 fileName
          • 0777 => drwxrwxrwx
          • 7777 => drwsrwsrwt
          • 7666 => drwSrwSrwT

        • Symbolic:

          • Pro of symbolic mode is that you can just change one section of permissions at a time instead of being forced to change them all with absolute mode!
          • Section Symbols:
            • "u": user owner
            • "g": group owner
            • "o": Other/everyone
            • "a": all of the above
              • If no section symbol is used, a is the default!
          • Operation Symbols:
            • "+": adding permissions
            • "-": subtracting permissions
            • "=": Instead of subtracting or adding, you absolutely set it equal to that permission. (u=r {user=read})
          • Use the section symbols to pick the owner section, then use operation symbols to give permissions.
            - This works for Special Permission too, but they require the section symbols to work.
            chmod symbolic mode
            • chmod u+rwx,g-rx,o=xt fileName

    • Hard Link Count: number of hard links with a default of 1.

    • User: Owner of file

    • Group: Group with access to file, Only one group at a time!

      • Use id -gn userName command to find the Primary group of a user which will be the group on there by default when a user creates a file.

    • File Size: size in bytes

    • Modification Time: date time of last modified.

    • Filename: name of file or folder.

Switching Users and Into Root

  • Use the su command which stands for switch user and NOT superuser.
  • This command is how you can switch to root.
    • su username (keeps the same shell)
    • su -l username (logs in as the user and uses there shell and env varibles)
    • su (will login you into root by default but without its shell)
    • su - (will login you into root but with its shell and is the same as using -l or --login)

Common Directories [NOT DONE]

  • /etc: etcetera
    • Stores system files for the operating system.
      • Such as the shadow file, passwd file, and sudoers file.
  • /var: variable data
    • Stores ramdom data for prorams and services.
      • Such as Logs, temp data, backups, and more.
  • /home
    • The directory that holds all the users home folders except for root.
  • /root
    • This is the home folder for the root user.
  • /tmp: Temporary
    - Volatile storage which means it erases when the computer restarts just like RAM.
    Feature

    Any User can write to the tmp directory by default.

Processes

  • Processes are units of work on linux and programs have many processes which are managed by the kernel.
    • They have an ID associated with it called PID (Process ID) which increments in the order at which processes start when the PC turns on.
    • You can see processes in the /proc directory
      Note
      • Processes are different than Jobs!
        • Jobs are not apart of the OS and use many programs executed as a shell command.
    • Linux uses namespaces to split up processes into sections which have certain access to resources.
    • The first process ran when a computer boots up is normally the init system such as (systemd).
      • Other processes ran afterwards are normally subprocesses of systemd.
  • ps: (Process Status) Display running processes and other Information
    • ps aux (all processes included those outside of the terminal and not owned by you)
    • ps -ux (processes you own and outside your terminal)
    • ps -U username (processes owned by that user)
    • ps -C programName (all processes for a given program)
    • ps -ux | grep regex (use regex to search processes)
      • Or use: pgrep regex
  • top: (Table Of Processes)
    • Use h key to show key binds.
    • Alternatives: btop, htop
  • uptime: Tells you how long the computer has been on.
  • kill: (Uses SIGTERM which terminates the program with cleanup by default)
    • kill PID (find PID uses pgrep and ps)
    • kill -[SIG] PID (give a different siginal with the SIG prefix)
      • Signals: TERM, KILL(Force w/o cleanup), STOP
    • kill $(pgrep programName)
  • pkill: (kill all processes for a given program)
    • pkill programName
  • xkill: (Interactive kill to choose which foreground process to kill)

Services

  • systemctl: (List all services and their statuses)
  • Format: sudo systemctl [option] serviceName
    • Options: enable, disable, start, stop, restart, reload, status

Cron Jobs

  • These are processes that are used to do specific tasks and actions such as running commands, backing up files, running programs, etc at a specific time that repeat over and over again.
    • Cron is the main process that gets called once a minute.
      • Cron checks all crontabs which are files in the /var/spool/cron directory
        • These files have cron jobs in them, one per line.
        • Blank lines and Commented (#) lines don't get read.
  • crontab: (command to edit your crontab file)
    • crontab -e (edit crontab file)
    • crontab -l (print out crontab file)
    • crontab -r (delete crontab file)
    • crontab fileName (make file your crontab file)
    • sudo crontab -u userName (superuser edits other users crontab files)
  • Job Syntax: Min Hour DayOfMonth Month DayOfWeek Command
    • * means the we dont care about that option. Aka it could be any of them.
    • MIN: {0-59} => (#) or (#-#) Range or (#,#) or (#-#,#-#) or [(*/n) or (#-#/n)] every nth min.
    • Hour: {0-23} => Same as MIN
    • DOM: {1-31} => Same as MIN
    • MONTH: {1-12} => Same as MIN and (jan, feb, mar, ... etc)
    • DOW: {0(Sun)-6(Sat)} => Same as MIN and (mon, tue, wed, ... etc)
    • COMMAND: any shell command
      • Use absolute paths for programs (/usr/bin/cmd) to ensure it works properly
      • You can use environment variables too!
        Predefined options
        • Instead of using the complicated syntax, there are a few predefined options:
          • @option:
            • yearly, annually, weekly, daily, midnight, hourly, minutely, secondly
